# UI-14 — Timesheet Draft Entry Surface

## Purpose

- Draft entry surface ONLY for snapshot-aligned fields
- Does NOT finalize, approve, or calculate money
- Feeds into existing UI-9/UI-10 approval flow

## Domain Locks

- UI-9 / UI-10 contract is FINAL — do not modify
- UI-14 drafts ONLY: `orderId`, `weekStart`, `workerId`, `totalHours`
- Forbidden: bonuses, adjustments, money/rate calculations, approval logic

## Endpoints

### GET `/ui/internal/ui14/timesheet-draft`

- Query: `orderId` (required), `weekStart` (required, ISO date)
- Returns: draft form HTML or current draft state

### POST `/ui/internal/ui14/timesheet-draft`

- Body: `{ orderId, weekStart, items: [{ workerId, totalHours }] }`
- Action: saves draft rows (no approval, no money calc)

## Allowed Files

- `src/ui/internal/routes/ui14.controller.ts`
- `src/ui/internal/routes/ui14.module.ts`
- `src/ui/internal/templates/ui14-timesheet-draft.html`
- `src/app.module.ts` — Ui14Module registration ONLY

## Forbidden Actions

- Modifying UI-9 or UI-10 files
- Changing Prisma schema or running migrations
- Adding bonus/adjustment/rate/money fields
- Touching any file not in Allowed Files list

## DONE Criteria

- `npm run check:bom` — passes
- Remove dist, then `npm run build:checked` — exits 0
- GET endpoint returns draft form
- POST endpoint saves draft (verify via DB or GET)
